# Ledgerly Design System

**Version:** 1.0

## Purpose

This document defines Ledgerly's visual language.

Every screen, component, and interaction must follow these guidelines to ensure a consistent experience.

---

# Design Principles

- Mobile First
- Calm Interfaces
- Minimal Visual Noise
- Accessibility by Default
- Consistency Over Creativity
- One Clear Primary Action Per Screen

---

# Color Philosophy

Color communicates meaning.

Never use color purely for decoration.

### Primary

Used for:

- Primary buttons
- Active navigation
- Focus states

### Success

Used for:

- Successful repayments
- Completed reminders
- Positive financial activities

### Warning

Used for:

- Upcoming due dates
- Partial repayments

### Danger

Used for:

- Overdue loans
- Validation errors
- Destructive actions

### Neutral

Used for:

- Text
- Borders
- Backgrounds
- Cards

---

# Typography

Hierarchy should communicate importance.

Levels:

- Display
- Heading
- Title
- Body
- Caption

Avoid unnecessary font-weight variation.

---

# Spacing

Ledgerly uses an 8-point spacing system.

Preferred spacing values:

- 4
- 8
- 16
- 24
- 32
- 48
- 64

Avoid arbitrary spacing.

---

# Border Radius

Use consistent rounded corners.

- Small
- Medium
- Large
- Extra Large

Cards, dialogs, and bottom sheets should share the same visual language.

---

# Shadows

Use subtle elevation.

Levels:

- Card
- Popover
- Dialog

Avoid heavy shadows.

---

# Icons

Use Lucide exclusively.

Icons should always accompany meaning, never replace it.

---

# Buttons

Supported variants:

- Primary
- Secondary
- Outline
- Ghost
- Destructive

Only one Primary button should exist on a screen whenever possible.

---

# Forms

Guidelines:

- Large touch targets
- Labels always visible
- Helpful validation messages
- Logical field grouping
- Smart defaults
- Progressive disclosure for optional fields

---

# Cards

Cards represent logical groups of information.

Typical card sections:

- Header
- Content
- Footer (optional)

Cards should remain lightweight and easy to scan.

---

# Lists

Lists are preferred over dense tables on mobile.

Each item should expose only the most important information, with secondary details available on the detail screen.

---

# Charts

Charts communicate trends, not raw data.

Requirements:

- Clear legends
- Minimal grid lines
- Accessible colors
- Responsive layouts
- Meaningful tooltips

---

# Motion

Animations should:

- Reinforce interactions
- Feel responsive
- Never delay the user

Use subtle transitions for:

- Navigation
- Dialogs
- Bottom sheets
- Chart updates

---

# Responsive Philosophy

Design starts with phones.

Tablet layouts expand.

Desktop layouts enhance.

Never design desktop first.

---

# Accessibility

Every interface should support:

- Keyboard navigation
- Screen readers
- High contrast
- Large touch targets
- Visible focus states

Accessibility is a requirement, not an enhancement.

---

# Design Goal

Users should immediately understand what to do without reading instructions.

Every interface should feel calm, trustworthy, and effortless.
